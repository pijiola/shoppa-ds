Meet Claude

Your AI design-to-code assistant

Claude is an AI that reads your Figma designs and writes the code for you. It understands your colors, spacing, fonts, and icons — and turns them into real, working components.

You don't need to write or understand any code. You just need to tell Claude what to do, in plain language, and review the result visually.

A Typical Session

Here's what it actually looks like

You open a conversation with Claude on your computer
You share a Figma link to the component you want to update
Claude reads the design and writes the code automatically
A live preview opens in your browser so you can check it
You say "yes, ship it" — and it's published
Visual Review

Iteration

What if the preview doesn't look right?

You click "No, it needs tweaks"
The changes are discarded — nothing broken, nothing shipped
You adjust the design in Figma
You share the updated Figma link
Claude tries again — as many times as needed
There's no risk. Nothing goes live until you explicitly approve it.
Collaboration

What about the developers?

Developers are still part of the process. Here's what changes for them:

  • They use your components instead of building their own
  • They can suggest changes — you review them visually
  • They focus on app features, not rebuilding UI elements
  • Fewer mismatches between design and what users see
This doesn't replace developers. It frees them to work on the things only they can do — while giving you control over the design details.

How to use it

A step-by-step guide. No coding knowledge needed.

One-Time Setup

Before your first time (1/2)

Someone from the team will help you set this up once. Here's what gets installed and why:

Claude Code
The AI assistant that reads your Figma designs and writes the code. It runs in the Terminal on your Mac.
→ docs.anthropic.com/claude-code
The design system project
All our components and their code, downloaded from GitHub to your computer. Think of it like syncing a shared Figma file, but for code.
→ github.com/pijiola/shoppa-ds
One-Time Setup

Before your first time (2/2)

Two commands get everything ready:

Install dependencies — npm install
Downloads everything the project needs to run. Like installing fonts before opening a file.
Start Storybook — npm run storybook
Launches the live preview tool in your browser. This is where you'll review components before approving them.
→ Public version: pijiola.github.io/shoppa-ds
This whole setup takes about 10 minutes and you only do it once. After that, you just open the Terminal and type claude.
Step 1

Open the Terminal

The Terminal is an app already on your Mac. Think of it as a text-based chat — you type, the computer responds.

Terminal
Open Spotlight (⌘ + Space)
Type Terminal
Press Enter
Step 2

Go to the project folder

Type this and press Enter. It tells the computer where the design system lives.

Terminal
$ cd ~/Dev/Design\ Systems/shoppa-ds
Tip: you can also drag the folder from Finder into the Terminal window after typing "cd ".
Step 3

Start Claude

Type this and press Enter. Claude will start and greet you.

Terminal
$ claude

╭──────────────────────────────────╮
│ ✦ Welcome to Claude Code! │
╰──────────────────────────────────╯

How can I help you today?
Step 4

Use a command + your Figma link

You have two commands. Pick the one that fits, paste your Figma link after it, and press Enter.

Update an existing component
> /update-component
https://figma.com/design/...

Reading the Figma design...
Updating the component...
✓ Opening Storybook
Create a brand new component
> /new-component
https://figma.com/design/...

Reading the Figma design...
Creating the component...
✓ Opening Storybook
Copy the Figma link from your browser's address bar and paste it after the command.
Step 5

Review and approve

Storybook opens in your browser automatically. Back in the terminal, Claude asks you a simple question:

Terminal — Claude Code
? Is the component good to go?

● Yes, commit and push
○ No, it needs tweaks
○ Other
Use the arrow keys (↑ ↓) to move between options and press Enter to confirm. Compare what you see in Storybook with your Figma design.

That's it.

Three commands to remember:

claude — to start a conversation
/update-component — to update an existing component
/new-component — to create a brand new one

Cheat Sheet

Keep this handy

Every time you start

1 Open Terminal (⌘ + Space → "Terminal")
2 Go to the project folder
3 Type claude and press Enter

To update a component

1 Copy the Figma link from your browser
2 Type /update-component + paste the link
3 Review in Storybook → Approve or reject

To add a new component

1 Copy the Figma link from your browser
2 Type /new-component + paste the link
3 Review in Storybook → Approve or reject

If something goes wrong

1 Don't panic — nothing ships without your approval
2 Click "No, it needs tweaks" to discard changes
3 Ask Claude in plain language what happened
